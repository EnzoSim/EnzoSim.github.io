FDA Catalyst Research

A research product for tracking biotech and pharma companies with near-term FDA or clinical catalysts. The current build turns stored BPIQ data into a compact live calendar with source links, event filters, market context, and TradingView links.

What exists now

The app has the core product surface live: a FastAPI service, Postgres-backed BPIQ extraction records, and a Next.js calendar page deployed on Railway.

API

FastAPI exposes the calendar payload through /bpiq/calendar, plus health, sources, catalysts, scanner, watchlist, backtest, and IV study endpoints.

Data

BPIQ REST API responses are stored as raw records locally and ported into Railway Postgres for the hosted calendar.

UI

The /calendar page shows dated catalysts, filter controls, source links, TradingView links, and a next-events rail.

Railway status

The Railway deployment is live with two services: a Python API and a Next.js web app. The current public URLs are generated Railway domains while custom subdomains are prepared.

Production check

The production smoke test loaded the live calendar with 128 visible rows, an API connection badge, PDUFA rows, no console errors, and no mobile horizontal overflow.

  1. API service: fda-catalyst-api, deployed from the repo root with a Python Dockerfile.
  2. Web service: fda-catalyst-web, deployed from apps/web with Node 20.
  3. Data: local BPIQ snapshot copied into Railway Postgres.
