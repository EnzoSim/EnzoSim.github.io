FDA Catalyst Research

A research product for tracking biotech and pharma companies with near-term FDA or clinical catalysts. The current build turns stored BPIQ data into a compact calendar with source links, event filters, market context, and a Railway deployment path.

What exists now

The local app already has the core product surface: a FastAPI service, SQLite-backed BPIQ extraction records, and a Next.js calendar page that can be promoted to a hosted deployment.

API

FastAPI exposes the calendar payload through /bpiq/calendar, plus health, sources, catalysts, scanner, watchlist, backtest, and IV study endpoints.

Data

BPIQ REST API responses are stored as raw records in SQLite locally, with a path to hosted Postgres for production.

UI

The /calendar page shows dated catalysts, filter controls, source links, TradingView links, and a next-events rail.

Deployment path

The intended production setup is two Railway services: one API service at the repo root, and one web service from apps/web. Custom domains should be attached after both generated Railway URLs pass smoke tests.

  1. Create or link the Railway project.
  2. Deploy the API service from the repo root.
  3. Deploy the web service from apps/web.
  4. Attach hosted Postgres through DATABASE_URL.
  5. Configure BPIQ and provider keys only as Railway variables.
  6. Attach catalysts.enzosimier.com and api-catalysts.enzosimier.com.
